### 상승 전 핵심 시그널 (2021년 중~2022년 초)

**시그널 1: 사업보고서 수주 현황 섹션의 급변**


수주잔고/매출 비율이 1.5배 → 2.8배로 급증할 때 향후 2년 매출이 확정적.

**시그널 2: 지정학 이벤트 + 섹터 연결**

- 2022년 7월: 폴란드 K-9 672문 + K-2 980대 + FA-50 48대 (약 17조원 규모)

뉴스에 "Poland", "NATO", "K-9", "방산 수출" 키워드가 공시 6개월 전부터 등장.

**시그널 3: 한화시스템·한화방산 분리 후 밸류에이션 재평가**

- 2022년: 방산 부문 한화에어로스페이스로 통합 → 수익성 높은 방산만 남음
- 건설·엔진 등 저마진 부문 분리 → OPM 급등 구조적 전환

**시그널 4: 분기별 수주 공시 패턴**

현대일렉트릭은 주요 수주 건별로 DART 공시 제출:

## 3. 효성중공업 (298040) — 동일 섹터, 다른 제품 mix

### 주가 궤적

| 시점 | 주가 | 사건 |
|---|---|---|

| 2022-10 | ~70,000원 | HVDC 수주 기대감 |
| 2023-06 | ~130,000원 | 건설/중공업 실적 개선 |
| 2024-06 | ~260,000원 | ~5배 상승 |

### 상승 전 핵심 시그널

**시그널 1: 수주잔고 구성 변화 (고마진→저마진 비율)**

효성중공업은 건설(저마진 5%)과 중공업/전력기기(고마진 10-15%)가 혼재.
2021-2022년 중공업 수주 비중이 급증하면서 전체 마진 상승 구조 형성.

사업보고서 사업부문별 수주잔고:
- 2020: 건설 70% / 중공업 30%
- 2021: 건설 55% / 중공업 45%
- 2022: 건설 40% / 중공업 60% ← 전환점

고마진 부문 비중이 50%를 넘으면 전사 영업이익률이 구조적으로 상승.

**시그널 2: HVDC·GIS 공시 키워드**

- HVDC(High Voltage Direct Current): 장거리 전력 전송, 해상풍력 연결
- GIS(Gas Insulated Switchgear): 고전압 개폐기

이 키워드들이 DART 공시에 등장하기 시작하면 고마진 수주 증가 시그널.

**시그널 3: 수출 비중**

2022년 사업보고서:
- 국내 수출: 40% → 55% (수출 비중 급증)
- 주요 수출국: 미국, 사우디아라비아, 인도

수출 비중이 50% 넘으면 글로벌 수요 직접 수혜 구조.

## 4. 세 종목의 공통 패턴 — "수주잔고 선행 법칙"

```
수주잔고 급증 (T)
    │
    │ +6개월
    ▼
매출 인식 시작 (T+6)
    │
    │ +6개월
    ▼
영업이익률 급등 (T+12)
    │
    │ +6개월
    ▼
시장 인식 / 주가 상승 (T+18)
```

**핵심 통찰**: 수주잔고가 급증할 때 주가는 아직 반응 안 한다.
매출이 실제로 올라가야 시장이 인식한다.
우리 시스템은 **수주잔고 단계에서** 포착해야 18개월 빨리 진입할 수 있다.

## 6. 추가해야 할 알고리즘 요소

### 요소 A: 수주잔고 파워 지표 (가장 중요)

**f13: 수주잔고/매출 비율 (Backlog Coverage Ratio)**

```python
# filters_kr.py에 추가
backlog = stock_data.get("order_backlog")
rev_ttm = stock_data.get("revenue_ttm")
if backlog is not None and rev_ttm is not None and rev_ttm > 0:
    bcr = backlog / rev_ttm  # Backlog Coverage Ratio
    if bcr >= 1.5:    # 1.5배 이상: 18개월치 수주 확보
        s = min(10, (bcr - 1.0) * 4)
        score += s
        result.scores_by_filter["f13_backlog_coverage"] = s
    if bcr >= 2.0:    # 2배 이상: 강력 시그널
        score += 5    # 추가 보너스
        result.scores_by_filter["f13_backlog_bonus"] = 5.0
```

**f14: 수주잔고 YoY 성장률**

```python
backlog_prev = stock_data.get("order_backlog_prev")
if backlog is not None and backlog_prev is not None and backlog_prev > 0:
    bg_growth = (backlog - backlog_prev) / backlog_prev * 100
    if bg_growth >= 30:  # 수주잔고 30%+ 증가
        s = min(12, (bg_growth - 20) / 5)
        score += s
        result.scores_by_filter["f14_backlog_growth"] = s
```

필요 데이터: `financials_dart.py`에서 사업보고서 수주현황 파싱 추가.

### 요소 B: OPM 전환점 감지

현재 f05는 OPM > 10%일 때만 점수를 준다.
이미 10% 넘었으면 주가는 이미 많이 올랐을 가능성이 높다.

**f15: 영업이익률 방향성 (저OPM에서 상승 중)**

```python
# OPM 전환점: -5% ~ +5% 구간에서 개선 중일 때 최고 점수
op_margin = stock_data.get("op_margin_ttm")
op_margin_prev = stock_data.get("op_margin_prev")
if op_margin is not None and op_margin_prev is not None:
    opm_delta = op_margin - op_margin_prev
    
    # 흑자 전환 (음→양)
    if op_margin_prev < 0 and op_margin > 0:
        score += 10  # 최고 시그널
        result.scores_by_filter["f15_turnaround"] = 10.0
    
    # 저마진 구간(0~5%)에서 개선 중
    elif 0 <= op_margin <= 7 and opm_delta > 1.5:
        s = min(8, opm_delta * 2)
        score += s
        result.scores_by_filter["f15_margin_inflection"] = s
```

### 요소 C: DART 사업보고서 파싱 확장

현재 `financials_dart.py`는 재무제표(손익·재무상태)만 파싱.
사업보고서의 "사업 내용" 섹션에는 더 중요한 데이터가 있다.

**추가 파싱 대상**:

```
사업보고서 > 2. 사업의 내용 > 나. 수주 현황
  ┌─────────────────────────────────────────────┐
  │ 품목    수주일    납기    수주총액    수주잔고  │
  │ K-9    2021.07   2027   12,000억   8,500억   │
  │ 변압기  2022.03   2025   3,200억    2,100억   │
  └─────────────────────────────────────────────┘
  합계: 수주총액 15,200억 / 수주잔고 10,600억
```

파싱 방법:
1. `dart.document(cno)` 또는 `dart.report()` API로 HTML 원문 취득
2. "수주 현황" 또는 "수주잔고" 테이블을 BeautifulSoup으로 파싱
3. 합계 행에서 수주잔고 총액 추출
4. 전기 대비 증가율 계산

### 요소 D: 공시 빈도 추적

단일 수주 공시 빈도가 증가하면 수주 momentum 시그널.

```python
# filings_watch.py 또는 새로운 scoring 함수
def score_filing_frequency(supabase_client, ticker: str, days: int = 90) -> float:
    """최근 N일 수주 관련 공시 빈도 점수."""
    cutoff = (date.today() - timedelta(days=days)).isoformat()
    res = supabase_client.table("filings")\
        .select("id")\
        .eq("ticker", ticker)\
        .gte("filed_at", cutoff)\
        .in_("keywords", ["수주", "계약", "수주잔고"])\
        .execute()
    
    count = len(res.data or [])
    # 분기 내 수주 공시 3건 이상 = 강력 시그널
    if count >= 6:  return 10.0
    elif count >= 3: return 6.0
    elif count >= 1: return 3.0
    return 0.0
```
