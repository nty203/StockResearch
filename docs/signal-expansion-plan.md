## 1. 문제 정의

기존 알고리즘은 **매출 성장 + 영업이익률** 중심으로 설계되어 있어, 다음 두 가지 유형의 10배 주식을 사전에 포착할 수 없다.

| 유형 | 대표 종목 | 왜 놓치는가 |
|---|---|---|
| **수주잔고 선행형** (매출 발생 중) | 한화에어로, HD현대일렉, 효성중공업 | 수주 폭발이 매출로 반영되기까지 9개월 지연 — 알고리즘이 지연된 신호를 본다 |
| **플랫폼 채택형** (매출 극소) | 로보티즈, 레인보우로보틱스, 펩트론, 삼천당제약, 대한광통신, 우리기술 | 손익계산서에 아무 것도 없는 상태에서 지분투자·임상·인증 공시가 선행 신호 |

### 2-1. 공통 패턴

| 종목 | 수주잔고 급증 시점 | 주가 반응 시점 | 선행 기간 |
|---|---|---|---|
| 한화에어로스페이스 | 2021 Q4 (폴란드 K-9 수출) | 2022 Q2+ | ~9개월 |
| HD현대일렉트릭 | 2022 Q3 (HVDC 변압기 수주) | 2023 Q2+ | ~9개월 |
| 효성중공업 | 2022 Q1 (미국 전력망 발주) | 2022 Q4+ | ~6개월 |

### 2-2. 구현한 필터 (f13/f14/f15)

```python
# f13: BCR (Backlog Coverage Ratio = order_backlog / revenue_ttm)
# BCR >= 0.5 = 향후 6~12개월 성장 가시성 확보
# BCR 0.5x → 2.5점, 2x → 10점

# f14: Backlog YoY growth >= 20%
# 수주잔고가 전년 대비 급증 = 미래 수주 가속 신호

# f15: OPM inflection (낮은 OPM에서 상승)
# op_margin_prev < 10% AND 개선폭 >= 1pp = 수익성 전환 초입
```

### 2-3. 기대 탐지율 개선

|---|---|---|
| 한화에어로스페이스 | 탈락 (수주 미반영) | 통과 (f13 BCR + f14 수주 급증) |
| HD현대일렉트릭 | 탈락 (저마진 시절) | 통과 (f14 + f15 OPM 반등) |

## 3. 6종목 케이스: 플랫폼 채택형 신호

### 3-1. 종목별 선행 신호 요약

| 종목 | 섹터 | 핵심 선행 신호 | 매출 발생 이전 기간 |
|---|---|---|---|
| 로보티즈 (108490) | 로봇 액추에이터 | LG전자 전략적 지분투자 + Dynamixel 학술 채택률 급증 | 18~36개월 |
| 대한광통신 (010170) | 광섬유·광케이블 | 수직계열화 완성 공시 + AI DC 규격 제품 개발 완료 | 24개월 |
| 삼천당제약 (000250) | 경구형 GLP-1 | 일본 기술이전 계약 → 유럽 라이선스 → 미국 본계약 | 24개월 |
| 레인보우로보틱스 (277810) | 협동로봇/휴머노이드 | 삼성 유상증자 참여 공시 + **콜옵션 조항** | 0일 (공시 당일) |
| 펩트론 (087010) | DDS + GLP-1 CDMO | FDA IND 승인 + 신공장 건설 + LG화학 유통 계약 | 18개월 |
| 우리기술 (032820) | 원전 MMIS | 원안위 인증 갱신 + 수주잔고/매출 배율 + 탈원전→원전 정책 전환 | 12개월 |

### 3-2. 공통 선행 신호 5개 범주

```
1. 전략적 지분투자 + 콜옵션 구조
   - 단순 재무투자 vs. 콜옵션 포함 전략투자 구분 필수
   - 콜옵션 = 대형 파트너가 향후 지배구조 변화를 염두에 둠

2. 임상 단계 진행 (바이오)
   IND 제출 → IND 승인 → 임상 완료 → 품목허가 → 출시
   - 각 단계 전환이 catalyst

3. 플랫폼 기술 인증 (로봇/원전)
   - 원안위 인증 갱신: 원전 독점 지위 유지 확인
   - 학술 논문 채택: 연구 → 상업화 선행

4. 생산능력 확장 공시 (신공장/신라인)
   - CDMO·원전 모두 공장 증설이 수주 확보 전제

5. 물질특허 만료 일정 (제약)
   - 오리지널 만료 시점이 제네릭·개량형 진입 촉진
```

### 4-2. 스크리닝 엔진 (filters_kr.py)

- size_score: avg_daily_value 기반 (기존: 항상 5점 고정 버그 수정)

|---|---|---|
| revenue_2y_ago | 항상 None | fins[8:12] 합산 |
| safety | debt_penalty 키 없어 항상 5 | debt_ratio → safety_score 참조 |
| size | f01 키 없어 항상 5 | avg_daily_value → size_score 참조 |
| quality | f05_margin_trend 누락 | 추가 |
